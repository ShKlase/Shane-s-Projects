Project Name: Pomodoro Timer App

Tech Used: HTML, CSS, JavaScript, Bootstrap

Description:
A feature-rich Pomodoro Timer web app designed to boost productivity by using the Pomodoro Technique. Users can set customizable work intervals, short and long breaks, and personalize the app with different fonts and colors. The timer includes visual progress indicators, pause/resume functionality, and an alarm for when a session ends.This app focuses on user experience and flexibility, making it ideal for developers who want a practical, interactive timer while practicing frontend skills.

Key Features & Learnings
 - Customizable Timer: Set Pomodoro, short break, and long break durations.
 - Pause & Restart: Users can pause/resume and reset the timer anytime.
 - Visual Progress Indicator: Circular SVG progress bar updates dynamically as time elapses.
 - Alarm Notifications: Audio plays when a session completes, with looping for continuous alert.
 - Theme Customization: Change app colors and fonts directly in the settings modal.
 - Responsive Design: Optimized for desktop and mobile using CSS and Bootstrap.
 - Settings Modal: Real-time input validation ensures users set time durations under 100 minutes.
 - DOM Manipulation Practice: Event listeners for buttons, inputs, and UI elements.
 - Local State Management: Tracks current session and updates UI dynamically.

Learned: Advanced JavaScript concepts like setInterval, clearInterval, SVG manipulation, dynamic DOM updates, event delegation, and modular UI interaction handling.

How to Use
1. Open index.html in a modern browser.
2. Select a timer type: Pomodoro, Short Break, or Long Break.
3. Start the timer by clicking the START button. You can pause/resume anytime.
4. Customize settings by clicking the gear icon:
5. Adjust timer durations (under 100 minutes).
6. Change font styles for the app.
7. Change accent color for progress circle and active timer buttons.
8. Click Apply in settings to save changes.
9. Press Restart to reset the current session.

Audio alarm plays at the end of the session.
Works offline with fallback offline page.

Tip: Always check that the timer durations are valid to prevent errors.
